LSTM

Long Short-Term Memory (LSTM)

提出

避免梯度爆炸 / 消失问题

结构

采用不同的路径处理长期和短期记忆
采用 Sigmoid 激活函数 和 Tanh 激活函数
Sigmoid 激活函数：将任意x轴坐标转换为y轴上0-1的数
$$f(x)=\frac{\mathrm{e}^x}{\mathrm{e}^x+1}$$
Tanh 激活函数：将任意x轴坐标转换为y轴上-1-1的数
$$f(x)=\frac{e^{x}-e^{-x}}{e^{x}+e^{-x}}$$

Green line: 被称为细胞状态，表示长期记忆。没有权重和偏差可以直接修改它，所以不会导致梯度爆炸或消失。
Pink line：被称为隐藏状态，表示短期记忆。短期记忆可以直接连接到可以修改的权重。

Forget Gate 遗忘门

input小 input大

减少了长期记忆。input很大时，长期记忆被完全遗忘。
蓝色框内的输出决定了记住长期记忆的百分比。

Input Gate 输入门

橙色部分：将短期记忆和输入组合，创建潜在的长期记忆。
绿色部分：确定将这个潜在记忆中的哪个百分比加入到长期记忆中
用短期记忆和当前输入来决定长期记忆忘记多少？记住什么？

经过输入门之后，我们得到一个新的长期记忆。

Output Gate

最终阶段更新短期记忆 Short-Term Memory

这个新的短期记忆0.98也是整个LSTM单元的输出

权重和偏置开始是随机初始化，通过训练来学习的。LSTM共享权重和偏差，每个输入的W和b均相同。为了能够处理不同长度的数据序列。

Word2vec模型

词嵌入

文字 --> 数字，相似的词用相近的数字表示。
在不同语境，给一个词分配多个数字，工作量很大

如图所示，"Troll 2"和"Gymkata"并不接近。 但是这两个词在训练数据中出现在相同的上下文中。反向传播会使它们的权重变得更加相似。
有多个分类输出 --> 使用交叉熵损失函数进行反向传播
从输入到激活函数的连接上的新权重就是词嵌入。

预测下一个词的表现也较好

Word2vec词嵌入

引出

  • 不只是随机分配数字给词汇 --> 训练一个相对简单的神经网络来为我们分配数字
  • NN优势：可以使用训练数据集中词汇的上下文，来优化用于嵌入的权重(就是激活函数前的权重) --> 可以使相似的词汇最终具有相似的嵌入 -->训练一个处理语言的神经网络更容易(学习一个词的使用有助于学习如何使用相似的词) --> 仅仅预测下一个词，并没有提供足够的上下文来理解每个词
  • Word2vec 是一种流行的词嵌入方法，可以用于包含更多的上下文
  • 通常不是仅使用两个激活函数来为每个词语创建两个词嵌入 --> 通常采用100个或者更多的激活函数来为每个词创建大量的嵌入

连续词袋 Continuous Bag of words

通过使用周围的词来预测中间发生的词来增加语境

跳跃模型 SkipGram

通过使用中间的词来预测周围发生的词来增加语境

负采样优化 Negative Sampling

通过随机选择一部分单词工作

Seq2seq

sequence-to-sequence 序列to序列
将一种语言翻译成另外一种
用不同长度的句子作为输出 --> 生成不同长度的句子作为输出
LSTM可以处理不同长度的输入和输出
词汇表中的各个元素叫做tokens

有了输入的嵌入层词汇表

Encoder

采用独热编码 one-hot 作为LSTM的输入

在实践中，为了有更多的权重和偏差，使模型更适合我们的数据，人们经常在输入中添加额外的LSTM单元，为了简化，在第一个阶段添加一个额外的LSTM单元

2 单词的嵌入值用作两个不同LSTM单元的输入值。这两个LSTM有自己单独的权重和偏差集。

这里 2 单词的嵌入值相同吗

在编码器中再添加一层LSTM层，Layer 1 的输出 是短期记忆或隐藏层，用作第二层的输入。

再上图中，我们有2个LSTM层，每层有2个LSTM单元。
Encoder 对输入句子进行编码，变成长期集合和短期记忆的集合(也称为细胞状态和隐藏状态的集合)。 --> Context Vector 上下文向量
然后我们需要解码 Context Vector

Decoder

Decoder 和 Encoder中的 LSTM 不同，有自己独立的权重和偏差。
Context Vector 被用与初始化 Decoder 中 LSTM 的隐藏状态单元
Decoder 的用途是解码 Context Vector 为输出句子
Encoder 和 Decoder 的 嵌入层如图所示：
每个 Token 有不同的嵌入值，由于不同的输入单词、样本、标记、权重

Decoder从句子末尾<EOS>开始。
Decoder 直到下一个的预测输出是<EOS>token，解码器才会停止。

全连接层

LSTM顶层的输出值通过额外的权重和偏差 转换 到全连接层(只是普通NN的另一个名字)中。

解码器阶段：由编码器展开的LSTM单元的两层创建的上下文向量，用于初始化解码器中的LSTM. Decoder 的输入来自以<EOS>开始的输入词嵌入层(Embedding layer)，但是随后输出由最后一层确定。实际上，解码器会继续预测单词，直到它预测出<EOS> token，或者达到某个最大输出长度。

Seq2seq Encoder-Decoder Model

参考Torch实现
通过将 Encoder 和 Decoder 解耦，输入文本和翻译后的输出文本，可以有不同的长度。
所有的权重和偏差通过反向传播训练。
